Webhook Service
O Webhook é a funcionalidade da API que possibilita a automatização do envio de dados e atividades de contatos do RD Station Marketing e CRM para sistemas externos. Assim, é possível que você utilize o seu próprio aplicativo personalizado para ler, salvar e realizar ações com esses dados. Esta é uma opção poderosa que permite manter todos os seus dados em sincronia e abre a possibilidade para todos os tipos de integração.

Sempre que o evento configurado como gatilho no Webhook é acionado, o RD Station Marketing fará o disparo de um LEAD e o RD Station CRM fará o disparo de uma Negociação para integrações externas, contendo o payload padrão em JSON ("pacote de dados") único e imutável para a URL de destino, que vai vai conter todas as informações disponíveis no Perfil do Lead ou todas as informações da Negociação, o que inclui todos os campos padrão e personalizados que foram preenchidos.

Para informações sobre autenticação (OAUTH) para uso das APIs de Webhooks, consulte a seção de Autenticação da API .

Eventos que acionam o envio dos Webhooks
O evento que vai acionar o envio do payload conforme descrição abaixo:

Eventos do RD Station Marketing:
Conversão: envia os leads que realizaram alguma conversão no RD Station Marketing. Você pode selecionar quais serão essas conversões ou deixar em branco. Neste caso, todo lead que realizar uma conversão será enviado. Se as conversões forem realizadas em uma Landing Page, Formulário ou Pop-up do RD Station Marketing, o identificador vai ser o título da página/nome do elemento criado; se for integrado (formulários integrados, API, etc), eles podem ser identificados pelo identificador da conversão;
Oportunidade: envia os leads marcados como Oportunidade (estrelinha). Exemplo: enviar o lead do RD Station Marketing para a ferramenta apenas quando ele for marcado como oportunidade manualmente, por integração ou por um fluxo de automação.
📘
O envio será acionado sempre que o contato cumprir o gatilho configurado no Webhook.
Por exemplo, se o gatilho é "conversão" em qualquer evento,o contato será enviado mais de uma vez; sempre que ocorrer a conversão.

Evento do RD Station CRM:
Negociação: envia informações sobre uma negociação presente no CRM. Exemplo.: Se ocorreu uma criação de negociação, atualização de informações ou deleção da negociação.
🚧
Nota: Para utilizar o evento de negociação é necessário ter uma conta no RD Station CRM e ser assinante do plano Basic ou PRO.
📘
O tipo de evento que aciona a solicitação de webhook é definido pelo campo "entity_type".
Segue a lista de tipos de eventos suportados:
"WEBHOOK.CONVERTED", "WEBHOOK.MARKED_OPPORTUNITY"
"crm_deal_created", "crm_deal_updated", "crm_deal_deleted"

Requisitos para funcionamento dos Webhooks
Os Webhooks criados devem obedecer aos seguintes requisitos:

Ser capaz de receber e consumir dados em formato JSON;
Possuir certificado de segurança;
Ter compatibilidade para fazer a leitura do pacote de dados que o RD Station envia;
Estar funcional, respondendo requisições POST com status 2.xx.
⚠️
Importações e atualizações manuais do contato não acionam os Webhooks.
Autenticação
Webhooks do RD Station CRM oferecem suporte a autenticação por cabeçalho HTTP customizado, permitindo que integrações externas validem a origem das requisições recebidas.

O RD Station CRM incluirá automaticamente o cabeçalho configurado em todas as requisições disparadas pelo webhook quando forem informados os parâmetros:

auth_header: nome do cabeçalho HTTP que será enviado na requisição.

auth_key: valor que será atribuído ao cabeçalho definido em auth_header.

Caso os parâmetros de autenticação não sejam informados, o comportamento do webhook permanece o mesmo, sendo enviada apenas a requisição padrão com Content-Type: application/json.

Retries Logic
Caso nosso serviço de webhooks tenha problemas ao entregar suas notificações, nós tentaremos enviá-las novamente 3 vezes.

Os possíveis problemas na entrega das notificações são:

Se seu endpoint de callback demorar mais de 5 segundos para responder.
Se a resposta do seu endpoint de callback tiver um status code diferente de 2xx.
Após a falha no envio acontecer, as notificações entram em uma fila para serem re-processadas. Caso o re-envio de uma notificação falhe 3 vezes seguidas, a notificação será marcada como falha e não será re-processada.

Seguindo essa lógica, se as notificações estejam chegando duplicadamente no seu sistema, é um indício de que o tempo de resposta do endpoint é maior do que 5 segundos.

Boas Práticas
Integrar diferentes produtos, como RD Station CRM e RD Station Marketing, é uma prática valiosa para otimizar processos e obter uma visão mais abrangente sobre seus dados e operações.

Para garantir uma integração eficiente e robusta, siga estas boas práticas:

Entendimento dos Produtos: Antes de começar a integração, familiarize-se com os produtos RD Station CRM e RD Station Marketing. Compreender suas funcionalidades, terminologias e fluxos de trabalho é crucial para uma integração bem-sucedida.

RD Station Marketing
RD Station CRM
Definição de Objetivos: Estabeleça claramente os objetivos da integração. Identifique quais dados e funcionalidades precisam ser sincronizados entre os produtos. Isso ajudará a direcionar o desenvolvimento e a testar a integração de maneira mais eficaz.

Ideias de casos de uso que podem ser desenvolvidos
Utilização de APIs: Ambos os produtos oferecem APIs robustas. Certifique-se de revisar a documentação oficial das APIs do RD Station CRM e RD Station Marketing. Planeje como você usará essas APIs para garantir uma comunicação eficiente entre os sistemas.

Autenticação Segura: Implemente uma autenticação segura para acessar as APIs. Utilize tokens de acesso e siga as práticas recomendadas de segurança para proteger as credenciais e dados sensíveis durante a comunicação entre sistemas.

Tratamento de Erros: Antecipe e gerencie possíveis cenários de erro. Implemente um tratamento adequado para erros de rede, timeouts, respostas inesperadas e situações em que os dados não podem ser sincronizados. Forneça mensagens de erro claras para facilitar a resolução de problemas.

Mapeamento de Dados: Realize um mapeamento completo dos dados entre os produtos. Certifique-se de que os campos de dados são consistentes em ambas as plataformas e considere a possibilidade de transformações de dados, se necessário.

Sincronização Incremental: Ao sincronizar dados, implemente uma lógica de sincronização incremental para reduzir a carga no sistema. Isso envolve identificar e atualizar apenas os dados que foram modificados desde a última sincronização.

Logs e Monitoramento: Implemente logs detalhados para rastrear atividades e identificar problemas rapidamente. Configure um sistema de monitoramento para alertar sobre falhas de integração ou comportamentos inesperados.

Testes: Conduza testes abrangentes em todos os cenários possíveis. Isso inclui testes de integração, testes de unidade e testes de aceitação do usuário. Certifique-se de testar a integração em ambientes de teste e produção.

Exemplos de casos de uso
Separamos alguns casos de uso que podem ser implementados para te ajudar a planejar o desenvolvimento de sua integração

Esses exemplos de casos de uso ilustram como a integração multiproduto entre o RD Station CRM e o RD Station Marketing pode melhorar a eficiência operacional e oferecer uma visão geral do ciclo de vida do cliente, desde a sua aquisição até a conversão.

Sincronização de Contatos:
Implemente uma rotina de sincronização diária para enviar novos contatos do RD Station CRM para o RD Station Marketing.
Atualize automaticamente os dados de contato no RD Station CRM quando um contato for atualizado no RD Station Marketing.
Objetivo: Manter os contatos atualizados entre o RD Station CRM e o RD Station Marketing.

Rastreamento de Oportunidades:
Quando uma oportunidade é ganha no RD Station CRM, acione automaticamente uma campanha correspondente no RD Station Marketing para enviar e-mails de agradecimento.
Atualize o status da oportunidade no RD Station CRM com base nas interações do lead com as campanhas de marketing.
Objetivo: Acompanhar o progresso das oportunidades e campanhas em ambos os produtos.

Segmentação Dinâmica:
Utilize dados do RD Station CRM, como o estágio de compra do lead, para criar segmentos dinâmicos no RD Station Marketing.
Atualize automaticamente os segmentos à medida que os leads avançam ou retrocedem nos estágios de compra no RD Station CRM.
Objetivo: Criar segmentos dinâmicos com base em dados de ambos os produtos para campanhas personalizadas.

Relatório Integrado de Desempenho:
Crie um relatório consolidado que inclua métricas do RD Station CRM, como taxas de conversão de oportunidades, e do RD Station Marketing, como taxas de abertura de e-mails.
Atualize automaticamente o relatório conforme novos dados são gerados em ambas as plataformas.
Objetivo: Consolidar dados de desempenho de marketing e vendas em um único relatório.

Gestão de Eventos:
Ao cadastrar um novo participante de evento no RD Station CRM, adicione automaticamente esse contato a uma lista de inscritos em um evento no RD Station Marketing.
Atualize os dados do participante automaticamente em ambas as plataformas à medida que informações são atualizadas.
Objetivo: Sincronizar dados de participantes de eventos entre os produtos.
